Screenshots on demand

Console and network diagnostics

Direct Azure DevOps submission

How BugMe works

  1. Start capture

    Open BugMe and begin recording, or use Alt+Shift+R from any active tab.

  2. Reproduce naturally

    Perform the bug flow while BugMe logs actions and optional screenshots in the background.

  3. Stop and review

    End the session to review action count, screenshot count, and diagnostics before submission.

  4. Create the ADO bug

    Select project and paths, submit once, and share the generated work item URL immediately.

Built for teams already using Azure DevOps

  • Works with your existing ADO organization, project, area path, and iteration path setup
  • Uses PAT-based authentication with Work Items Read and Write scope
  • Keeps capture data local in browser storage until you choose to submit
